# Contributing to Azokle Auth

Thank you for your interest in contributing to Azokle Auth! We welcome bug reports, feature requests and pull requests.

## Reporting Issues

Before opening an issue, please:

1. **Search existing issues** to avoid duplicates.
2. **Provide clear steps to reproduce** if reporting a bug.
3. Include your **Android version**, **device model**, and **Azokle Auth version**.

## Submitting Pull Requests

1. **Fork** the repository on GitHub: [azoklesoftware/azokle-authenticator](https://github.com/azoklesoftware/azokle-authenticator)
2. **Create a branch** from `main` for your changes.
3. **Write clear commit messages** describing what and why (not just how).
4. **Test your changes** on a physical device or emulator before submitting.
5. **Open a pull request** against the `main` branch with a description of your changes.

## Code Style

- Follow the existing Java code style in the project.
- Keep changes focused and minimal — avoid unrelated refactors in the same PR.

## Security Issues

If you discover a security vulnerability, please **do not open a public issue**. Instead, contact us directly at [support@azokle.com](mailto:support@azokle.com).
